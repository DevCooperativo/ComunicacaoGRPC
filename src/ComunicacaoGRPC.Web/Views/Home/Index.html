Sobre o Sistema

Este sistema foi desenvolvido utilizando ASP.NET Core MVC no frontend e gRPC para comunicação com o backend. Isso garante uma comunicação extremamente rápida, eficiente e com baixo consumo de banda — ideal para aplicações modernas que exigem alta performance.

Como o gRPC é utilizado aqui?
  • O frontend envia requisições gRPC (através de clients gerados automaticamente).
  • O backend expõe serviços como: CompetiçãoService e ApostaService.
  • As mensagens trafegam usando Protocol Buffers (.proto), garantindo alto desempenho.
  • Os dados de competições e apostas são enviados e recebidos via contratos fortemente tipados definidos nos arquivos .proto.
Arquitetura simplificada

O sistema segue esta estrutura:

  • Projeto gRPC — controla regras de negócio e acesso ao banco.
  • Projeto MVC — renderiza as páginas e consome o serviço gRPC.
  • Entity Framework Core — para persistência de dados.
